# LX浏览器选择器

LX浏览器选择器是一个用于 Windows 的多浏览器选择器。它可以作为默认浏览器候选接管系统发往默认浏览器的打开请求，让你在打开网页链接、HTML 文件或其他浏览器唤起场景时选择具体浏览器，并按来源保存规则。

## 功能

- 带浏览器图标的选择窗口
- 仅一次 / 始终使用
- 全局默认浏览器
- 按来源应用保存规则
- 每次询问规则
- 手动管理浏览器路径
- 扫描候选浏览器
- 内置英文和简体中文界面
- 首次启动引导 Windows 默认应用设置
- 通过托盘监听和全局快捷键打开任意应用中的选中文本链接
- 内置更新检查，支持 Gitee / GitHub / jsDelivr 元数据回退
- 默认浏览器保护，检测系统默认浏览器是否被其他程序接管
- 关于页显示项目信息和可选支持说明

## 安装

运行 `LXBrowserPickerSetup.exe`，选择语言和安装路径，然后在安装完成页启动程序。

首次启动时，程序会引导你打开 Windows 默认应用设置。请把 `http` 和 `https` 设置为 `LXBrowserPicker`。

Windows 不允许应用静默把自己设为默认浏览器，所以这一步必须手动完成。

## 使用

- `仅一次`：只打开本次链接。
- `始终`：保存当前来源应用的浏览器规则，并打开链接。
- `设置`：管理浏览器、规则、全局默认浏览器和语言。

选词打开链接可以在“设置 -> 选词打开链接”中启用。它依赖托盘监听和可自定义的全局快捷键。`Ctrl+Alt+X` 会按规则和默认浏览器打开第一个识别到的链接。`Ctrl+Alt+C` 会每次手动选择浏览器；弹窗中单击浏览器图标即可打开。选词允许有多余内容。非链接内容不会搜索。

启用裸域名识别后，`example.com/path`、`example.cn` 这类常见域名会按链接处理，并自动补成 `https://...` 打开。

隐私说明：按下快捷键时，程序会临时复制选中文本并读取剪贴板，随后尝试恢复原剪贴板。

如果 Windows 通过系统进程中转启动选择器，程序可能无法识别真实来源应用；这种情况下，`始终` 可能保存为全局默认。

更新检查默认开启，但 LXBrowserPicker 不会自动下载或安装更新。它只检查版本元数据，并在你选择下载时打开官方 GitHub Release 页面。

默认浏览器保护会检查 Windows 是否仍把 `http` 和 `https` 交给 LXBrowserPicker。Windows 仍可能要求你在默认应用设置中手动确认；LXBrowserPicker 不会修改受保护的 `UserChoice` 注册表哈希。

## 配置

用户配置保存在 `%AppData%\LXBrowserPicker\lx-browser-picker.config.json`。

如果用户配置不存在，LXBrowserPicker 会读取程序同目录的 `lx-browser-picker.config.json` 作为安装默认模板。`lx-browser-picker.config.example.json` 只是模板。

## 旧版 BrowserPicker

LX浏览器选择器不会自动删除旧的 `CodexBrowserPicker` 注册表项。如果不再需要旧版，请单独卸载或清理。
